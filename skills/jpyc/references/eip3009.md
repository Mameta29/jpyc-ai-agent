# EIP-3009: Gasless Transfers

## transferWithAuthorization

```solidity
function transferWithAuthorization(
    address from,          // Payer (authorizer, signs the message)
    address to,            // Payee
    uint256 value,         // Amount to transfer
    uint256 validAfter,    // Unix timestamp: valid AFTER this time (use 0 for immediate)
    uint256 validBefore,   // Unix timestamp: valid BEFORE this time (expiry)
    bytes32 nonce,         // Random unique bytes32 â€” NOT sequential
    uint8 v,
    bytes32 r,
    bytes32 s
) external
```

- Caller can be **anyone** (relayer, payee, any third party)
- Reverts if: paused, `from` or `to` blocklisted, nonce used/canceled, time window invalid, signature invalid

## receiveWithAuthorization

```solidity
function receiveWithAuthorization(
    address from,
    address to,
    uint256 value,
    uint256 validAfter,
    uint256 validBefore,
    bytes32 nonce,
    uint8 v,
    bytes32 r,
    bytes32 s
) external
```

- **`msg.sender` must equal `to`** â€” prevents front-running; only the intended payee can submit
- Otherwise identical to `transferWithAuthorization`

## cancelAuthorization

```solidity
function cancelAuthorization(
    address authorizer,    // Must match signer
    bytes32 nonce,
    uint8 v,
    bytes32 r,
    bytes32 s
) external
```

- Marks the nonce as used without executing a transfer
- Only works if nonce is **not yet used**
- Reverts when paused

## authorizationState

```solidity
function authorizationState(address authorizer, bytes32 nonce)
    external view returns (bool)
```

Returns `true` if the nonce has been used or canceled.

## Signature Construction (TypeScript / viem)

```typescript
import { parseEther, toHex } from 'viem'
import { getAddress } from 'viem/utils'

const JPYC_ADDRESS = '0xE7C3D8C9a439feDe00D2600032D5dB0Be71C3c29' as const

// 1. Generate random nonce
const nonce = toHex(crypto.getRandomValues(new Uint8Array(32))) // bytes32

// 2. Build typed data
const validBefore = BigInt(Math.floor(Date.now() / 1000) + 3600) // 1hr

const typedData = {
  domain: {
    name: 'JPY Coin',
    version: '1',
    chainId: 1,                 // target chain
    verifyingContract: JPYC_ADDRESS,
  },
  types: {
    TransferWithAuthorization: [
      { name: 'from',        type: 'address' },
      { name: 'to',          type: 'address' },
      { name: 'value',       type: 'uint256' },
      { name: 'validAfter',  type: 'uint256' },
      { name: 'validBefore', type: 'uint256' },
      { name: 'nonce',       type: 'bytes32' },
    ],
  },
  primaryType: 'TransferWithAuthorization' as const,
  message: {
    from:        senderAddress,
    to:          recipientAddress,
    value:       parseEther('100'),
    validAfter:  0n,
    validBefore,
    nonce,
  },
}

// 3. Sign
const signature = await walletClient.signTypedData(typedData)

// 4. Decode v, r, s from signature
const r = `0x${signature.slice(2, 66)}` as `0x${string}`
const s = `0x${signature.slice(66, 130)}` as `0x${string}`
const v = parseInt(signature.slice(130, 132), 16)

// 5. Submit (anyone can call this)
await contract.write.transferWithAuthorization([
  senderAddress, recipientAddress, parseEther('100'),
  0n, validBefore, nonce, v, r, s
])
```

For `receiveWithAuthorization`, use type `ReceiveWithAuthorization` (same fields), and the payee must submit:
```typescript
// Only the payee (to) can call this
await contract.write.receiveWithAuthorization([...], { account: recipientAddress })
```

For `cancelAuthorization`:
```typescript
types: {
  CancelAuthorization: [
    { name: 'authorizer', type: 'address' },
    { name: 'nonce',      type: 'bytes32' },
  ],
}
```

## Nonce Strategy

EIP-3009 nonces are random `bytes32` â€” NOT sequential. Generate with `crypto.getRandomValues()` or `randomBytes(32)`. A nonce is invalidated once used or canceled. Check with `authorizationState(from, nonce)` before signing/submitting.
